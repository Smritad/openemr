Before you start

This guide walks through every top-level menu in the OpenEMR build that ships with the Matrix Bricks brand theme. For each tab you get:

  • What it is — a one-line definition.
  • Who uses it — the role that lives in that tab day-to-day.
  • Demo flow — the exact clicks to show during a walkthrough.
  • Talking point — a sentence to repeat to your boss / a prospect.

The screen mockups in this PDF are simplified illustrations of the live UI — they are designed to print cleanly. Show the live application alongside this guide for the full visual effect.

"Every clinic workflow has a tab. The doctor's day, the receptionist's day, the manager's day — each is a different vertical strip of this menu."

4. Recalls

What it is

Reminders for patients who need to come back — annual physical, vaccine, follow-up. Generates printable, e-mailable, or SMS-able lists.

Demo flow
  1. Open Recalls → filter by date range and recall type.
  2. Print, or export CSV for outbound calls / SMS blast.
"This is how a clinic keeps revenue predictable — no one falls through the cracks."

9. Modules

What it is

Pluggable add-ons. The stock build ships Manage Modules — install, enable, or disable extras such as FHIR API, OAuth2, CCDA, Direct Mail, e-prescribing, image viewers.

"OpenEMR is a platform. Regulatory and interop features ship as modules so we deliver them on the customer's schedule, not the vendor's."

12. Ensora eRx (Electronic Prescribing)

Direct connect to an e-prescribing network (NewCrop / DrFirst). Sends prescriptions electronically to the patient's pharmacy.

14. Reports

Read-only analytics and exports.

Top categories

  • Clinical — diagnoses, procedures, medications, immunisations, lab-result trends.
  • Visits — appointment volume, no-show rate, by-provider load.
  • Financial — Cash Rec · Front Rec · Pmt Method · Collections & Aging · Patient Ledger · Revenue by Service Code · Payment Processing.
  • Inventory — list, activity, transactions.
  • Procedures — pending results, pending follow-up, statistics.
  • Insurance — payer distribution, indigent care log, unique subscribers.
  • Statistics — clinic-wide KPIs.
  • Services / Background Services / Direct Message Log / IP Tracker — operational reports.
Demo flow
  1. Reports → Clinical → Diagnoses → see ICD-10 distribution across demo patients (I10, E11.9, M54.5, J02.9).
  2. Reports → Financial → Cash Rec → today's intake.
  3. Reports → Visits → appointment counts.
"Boss asks 'how was last month' → one click, dashboard answer."

16. Popups

Quick-action pop-out windows for the current patient — always one click away from the chart.

  • Issues — problem / allergy / medication list popup.
  • Export / Import — quick CCDA / PDF.
  • Appointments — book without leaving the chart.
  • Superbill — printable encounter charge sheet.
  • Payment / Checkout — quick capture.
  • Letter — referral / sick-note / school excuse from a template.
  • Chart / Barcode / Address labels — physical printing.
"All the small, frequent paperwork tasks — designed to take seconds, not minutes."

17. Cross-cutting features

  • Role-based access (ACL) — every menu item is gated by a role. A receptionist literally can't see the Reports tab if you don't want them to.
  • Audit log — every read and write is logged (HIPAA & India DPDP ready).
  • APIs — REST + FHIR + CCDA + HL7 → integrate with labs, HIEs, government registries, analytics.
  • Themes — fully re-skinnable. This build runs the Matrix Bricks brand SCSS (red / yellow / black).
  • Multi-language — UI labels live in a translation table. Ship in English + any regional language without touching code.
